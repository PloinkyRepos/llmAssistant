llmAssistant Agent Documentation

This repository currently hosts one Ploinky agent: llmAssistant.

The agent provides LLM-backed helper tools consumed by Explorer and by other agents for code completion, commit-message generation, and conflict resolution.

All MCP tool invocations execute through llmAssistant/tools/llm_tool.sh and llm_tool.mjs.

Operational Role

llmAssistant intermediates between caller requests and Achilles default LLM runtime execution by exposing a finite MCP contract from llmAssistant/mcp-config.json.

The agent normalizes MCP envelopes, validates input before model execution, and returns reusable helper outputs for editor and Git workflows without leaking provider-specific details into clients.

Repository Scope

The repository can host multiple Ploinky agents over time. The current scope covers llmAssistant as the dedicated LLM helper boundary for Explorer and agent-to-agent utility flows.
